# **前后端分离 + WebSocket 实时通信的极简栈**

## **架构说明文字（可直接写在 PPT 下方）**

### **✔ 架构模式：前后端完全分离**

- 前端（浏览器）负责 UI 展示、消息输入、本地搜索、用户列表等交互逻辑
- 后端（Node.js）提供静态资源 + WebSocket 服务
- 两者之间只通过 WebSocket 通信，耦合度极低、部署灵活

------

### **✔ 通信方式：WebSocket（全双工、长连接）**

- 每个客户端与服务器之间建立 **一条持久连接**
- 支持消息即时推送，不需要轮询
- 服务器收到消息后，通过广播模式同时推送给所有客户端

------

### **✔ 拓扑结构：标准“星型架构”**

- 所有客户端都连接同一个中心节点（服务器）
- 服务器负责消息中转、转发、同步状态
- 逻辑简单清晰、维护和调试成本低

------

### **✔ WebSocket 实时通信流程**

1. 客户端发起 WebSocket 连接
2. 连接建立后进入实时通信状态
3. 客户端发送消息 → 服务器接收
4. 服务器将消息广播给所有在线客户端
5. 客户端实时刷新 UI、展示消息
6. 客户端断开 → 服务器更新在线人数

------

### **✔ 架构优点总结**

- **实时性强**：毫秒级消息同步
- **实现简单**：无复杂协议，后台仅需广播
- **扩展容易**：可接入多客户端（PC、移动端均可）
- **调试成本低**：单节点架构逻辑清晰
- **资源占用少**：无数据库、无重型框架
